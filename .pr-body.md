## Problem

MaverickMCP currently hardcodes LLM provider endpoints and uses implicit auto-detection based on which API key is present. Users cannot:

- Point the app at a self-hosted OpenAI-compatible endpoint (LM Studio, Ollama, vLLM, LiteLLM, corporate proxy)
- Use a custom Anthropic proxy endpoint
- Override the OpenRouter base URL
- Explicitly select which provider to use (selection is implicit)
- Override default model names

## Solution

Add a `LLMSettings` Pydantic configuration model with environment variable support for BYOK-style provider + endpoint configuration.

### New Environment Variables

| Variable | Default | Purpose |
|---|---|---|
| `LLM_PROVIDER` | `auto` | Explicit provider: `auto`, `openrouter`, `openai`, `anthropic` |
| `OPENAI_BASE_URL` | _(none)_ | Custom OpenAI-compatible endpoint |
| `ANTHROPIC_BASE_URL` | _(none)_ | Custom Anthropic endpoint |
| `OPENROUTER_BASE_URL` | `https://openrouter.ai/api/v1` | Custom OpenRouter endpoint |
| `OPENAI_DEFAULT_MODEL` | `gpt-4o` | Override default OpenAI model |
| `ANTHROPIC_DEFAULT_MODEL` | `claude-sonnet-4-20250514` | Override default Anthropic model |
| `LLM_TEMPERATURE` | `0.3` | Override default temperature |

### Files Changed

| File | Change |
|---|---|
| `maverick_mcp/config/settings.py` | Add `LLMSettings` class, wire into `Settings` |
| `maverick_mcp/providers/llm_factory.py` | Read from `LLMSettings`, pass `base_url`, support explicit provider |
| `maverick_mcp/providers/openrouter_provider.py` | Make `base_url` configurable |
| `.env.example` | Document all new BYOK vars |
| `docs/CONFIGURATION.md` | BYOK usage guide with 8 examples |

### Backward Compatibility

All new environment variables are **optional** with safe defaults. When no new variables are set, behavior is identical to the current `main` branch.

### Testing

- 60 new tests (`test_llm_settings.py` + `test_llm_factory.py`) — all passing
- Existing test suite unaffected (all pre-existing tests pass)
- Lint and format clean on all changed files

### Example: Self-hosted OpenAI-compatible endpoint

```env
LLM_PROVIDER=openai
OPENAI_API_KEY=local-key
OPENAI_BASE_URL=http://localhost:1234/v1
OPENAI_DEFAULT_MODEL=llama3
```

### Example: Anthropic via corporate proxy

```env
LLM_PROVIDER=anthropic
ANTHROPIC_API_KEY=your-key
ANTHROPIC_BASE_URL=https://ai-proxy.internal.example.com
```
